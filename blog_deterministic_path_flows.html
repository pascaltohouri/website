Paths Through the Clock-Work Universe

Path Preferences

In the last blog post, we described a general autoregressive method for moving between experiments in the semantic vector space. This process lets us trace a path of experiments whose coordinates we could, at any time, decode to recover a full description of our machine learning setup. The autoregressive model was naive because it did not specify a preference over new potential experiments. Since we seek a specific type of experiment that maximises our utility (e.g., due to high performance, efficiency, or interpretability) we can use the laws of information to steer ourselves towards those optimal experiments in the SVS.

Path Preferences

If we let our imaginations run wild, we might imagine that the apparent complexity of the physical world is generated by a relatively small set of underlying mechanisms, like clockwork hidden behind the face of a watch. To understand the path, we would inspect the gears, which are the update rule. In this idealisation, the next experiment is determined. Our preferences would not alter the trajectory; at most, they would assign higher or lower value to points along it.

Viewed through the SVS lens, this is the claim that there exists an update rule (a ``law'') that maps the current experiment encoding to its successor, \[ X_{t+\Delta} = f(X_t), \] so the trace is a deterministic path. More generally, the laws of nature (physical, informational, and computational) restrict which moves are even feasible, confining us to a subregion of the SVS; in a fully determined universe, the feasible successor set collapses to a singleton - the unique next state compatible with the current state.

In discrete time, this ``law'' is a transition map \(f\). In continuous time, the same idea is expressed as a vector field \(v\) on the SVS, \(\frac{dX}{dt}=v(X)\). So, the laws of nature would be a vector field on the SVS, prescribing experiment state flows. Of course, this is a specific case of the more general universe-state flows that govern the movement of our universe across time and through SVS representations.

Semantic Vector Space Properties

Self-Reference

It is interesting to discuss the information that the semantic vector space (SVS) encodes. For example, a semantic vector X may encode a model’s parameters, or X may represent the architecture (full computational graph). X may represent the data and its generation process. So, can X represent the architecture and data at once? Can we continue, in a process of sequential enrichment, to encode more and more information into each point of the SVS? If so, we might imagine some semantic vector for every distinct object, and this conclusion has strange implications for the SVS’s properties.

Take self-reference, if X encodes all the features of a machine learning experiment, and if that experiment is a search algorithm over the semantic vector space, then X can access or refer to itself. This phenomenon is called self-reference, and generally, if a space contains all objects, then it must contain a description of itself (as the space is an object). To formalise, consider a vector space:

where a conceptual universe of distinct objects $o \in \mathcal{O}$ are described via an encoding $\phi$:

$$\phi: \mathcal{O} \to \mathbb{R}^N.$$

Then:

\[ \exists\, o \in \mathcal{O}\ \ \text{s.t.}\ \ o=\text{``a procedure that takes }\phi(o)\text{ as input.''} \]
