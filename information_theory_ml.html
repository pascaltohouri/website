Linguistic Descriptions of Experiment Traversals

Published 2025-11-18

Learning Representations and Symbolic Information

A machine learning experiment comprises many different elements. For example, when training an LLM, one specifies the architecture (attention heads, number of encoders and decoders, hyper-parameters/learning rate, etc.). These different elements are what make the experiment unique; changing these elements would recover a different experiment.

The symbolic information that each experiment contains is like an abstract symmetry preserved accross different representations. For example, one might represent an experiment (where an LLM is trianed via SGD) as a programming script, a semantic vector, or in natural language. Each representation has different linguistic rules, and therefore different properties (e.g., completeness, description lengths, self-referncing, etc.), but the essence of a given experiment is preserved when translating between representations.

From this view, every AI experiment has some essence, and a task with well-defined preferences over all symbolic experiments may have an optimum; a winner exists. With constant experiment features, the researcher may never find the winner. Sticking to an LLM architecture, for example, may guarantee that another superior architecture is never discovered, irrespective of the gains from scaling and training.

General Information-Theoretic Semantics

Experiments are not the only type of information that occupies this symbolic space of forms. Any piece of information, distinct from nothingness and from any other piece of information may inhabit the space. For example, a dog, a person, a house, a feeling, an image - all these different human comprehensible objects has a unique essence that presents differently depending on the language used. This seems obvious because a single object has a different name in each language (e.g., "dog" is equivalent to "perro", "hund", "skylos", "kutta", and "gou"). "
